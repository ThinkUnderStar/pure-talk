# PureTalk 前端技术文档

## 项目概述

PureTalk 是一个基于 Vue 3 + TypeScript + Vite 构建的前端项目，与后端 API 进行交互，实现了用户登录、注册、帖子浏览、评论、个人中心等功能。

## 技术栈

- **框架**: Vue 3
- **语言**: TypeScript
- **构建工具**: Vite
- **路由**: Vue Router
- **HTTP 客户端**: Axios
- **样式**: 原生 CSS

## 功能实现

### 1. 首页 (Home.vue)

- **功能**: 
  - 帖子列表展示
  - 分类切换
  - 下拉无限刷新
  - 帖子详情跳转

- **技术点**: 
  - 使用 `ref` 和 `onMounted` 管理组件状态和生命周期
  - 实现了下拉无限刷新功能，通过监听滚动事件触发加载更多
  - 使用 `debounce` 函数优化滚动事件处理，避免频繁触发

### 2. 登录页 (Login.vue)

- **功能**: 
  - 密码登录
  - 验证码登录
  - 发送验证码

- **技术点**: 
  - 使用 `ref` 管理表单状态
  - 实现了验证码倒计时功能
  - 使用 `debounce` 函数优化验证码发送按钮，避免重复点击

### 3. 注册页 (Register.vue)

- **功能**: 
  - 用户注册
  - 手机验证码发送和验证
  - 邮箱验证码发送和验证

- **技术点**: 
  - 实现了双验证码验证（手机和邮箱）
  - 使用 `debounce` 函数优化验证码发送按钮
  - 分步验证验证码，确保注册流程的正确性

### 4. 帖子详情页 (PostDetail.vue)

- **功能**: 
  - 帖子内容展示
  - 评论列表展示
  - 评论发布
  - 点赞功能
  - 评论下拉无限刷新

- **技术点**: 
  - 实现了评论的无限滚动加载
  - 实时更新点赞数
  - 使用 `debounce` 函数优化滚动事件处理

### 5. 个人中心页 (UserProfile.vue)

- **功能**: 
  - 用户信息更新
  - 密码修改
  - 账户删除
  - 登出

- **技术点**: 
  - 表单数据管理
  - 危险操作的确认提示

## API 接口

### 用户相关

- `POST /user/login` - 密码登录
- `GET /user/login/code` - 发送登录验证码
- `POST /user/login/code` - 验证码登录
- `POST /user/register` - 注册
- `GET /user/register/phone` - 发送注册手机验证码
- `POST /user/register/phone` - 验证手机验证码
- `GET /user/register/email` - 发送注册邮箱验证码
- `POST /user/register/email` - 验证邮箱验证码
- `DELETE /user/delete` - 删除账户
- `GET /user/logout` - 登出
- `PUT /user/update` - 更新用户信息

### 帖子相关

- `GET /post` - 获取帖子列表
- `POST /post/send` - 发送帖子
- `DELETE /post/user/{postId}/delete` - 删除自己的帖子
- `DELETE /post/admin/{postId}/delete` - 管理员删除帖子
- `PUT /post/{postId}/like` - 点赞帖子
- `PUT /post/view` - 更新浏览量

### 评论相关

- `GET /comment` - 获取评论列表
- `POST /comment/send` - 发送评论
- `DELETE /comment/user/{commentId}/delete` - 删除自己的评论
- `DELETE /comment/admin/{commentId}/delete` - 管理员删除评论
- `PUT /comment/{commentId}/like` - 点赞评论

### 通知相关

- `GET /notification` - 获取通知列表
- `GET /notification/{notificationId}/target` - 获取通知目标

## 技术亮点

1. **TypeScript 类型定义**: 为所有 API 接口和表单数据定义了清晰的类型，提高了代码的可维护性和可靠性。

2. **防抖优化**: 使用 `debounce` 函数优化了滚动事件和验证码发送按钮，减少了不必要的请求和事件触发。

3. **无限滚动加载**: 实现了帖子和评论的下拉无限刷新功能，提升了用户体验。

4. **响应式设计**: 适配了不同屏幕尺寸，确保在移动端和桌面端都有良好的显示效果。

5. **模块化结构**: 代码结构清晰，模块化程度高，便于后续扩展和维护。

## 注意事项

1. **后端 API 地址**: 前端通过 Vite 的代理配置将 `/api` 前缀的请求代理到 `http://localhost:8001`，请确保后端服务运行在该地址。

2. **登录状态管理**: 登录成功后，token 会存储在 localStorage 中，用于后续请求的身份验证。

3. **错误处理**: 前端对 API 请求的错误进行了基本处理，确保用户体验的流畅性。

4. **安全性**: 由于是前端项目，敏感操作（如账户删除）需要用户确认，以防止误操作。
